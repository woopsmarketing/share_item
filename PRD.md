# PRD (Product Requirements Document)

## 1. 배경 및 목적

여러 플랫폼·서비스를 개발할 때마다 비슷한 기능을 반복 개발하는 비효율이 발생한다.
**share_item**은 이 문제를 해결하기 위해, 공통 기능을 한 번 잘 만들어두고 다른 프로젝트에 복사·커스텀해서 재사용하는 모듈 저장소다.

---

## 2. 목표

- 플랫폼 간 공통 기능을 모듈 단위로 개발·보관
- 각 모듈은 복사 후 최소한의 수정으로 다른 프로젝트에 적용 가능해야 함
- 커스텀이 필요한 포인트는 명확하게 문서화

---

## 3. 타겟 사용자

- 이 레포를 관리하는 개발자 본인
- 복사한 모듈을 이어받아 커스텀하는 프로젝트 담당자 (본인 포함)

---

## 4. 모듈별 요구사항

---

### 4.1 AI 챗봇 위젯 (`ai-chatbot-widget`)

#### 개요
플랫폼에 삽입 가능한 24시간 AI 챗봇 위젯.
각 프로젝트마다 LLM이 알아야 할 정보(서비스 소개, FAQ, 운영 정책 등)가 다르기 때문에,
**RAG 구조로 설계하되 컨텍스트 소스만 교체하면 다른 프로젝트에 바로 적용 가능**하도록 만든다.

#### 기능 요구사항
- [ ] 화면 우하단 플로팅 버튼으로 위젯 열기/닫기
- [ ] 스트리밍 응답 지원 (타이핑 효과)
- [ ] RAG 파이프라인: 벡터 DB에서 관련 컨텍스트 검색 후 LLM에 주입
- [ ] 대화 히스토리 유지 (세션 단위)
- [ ] 모바일 반응형

#### 커스텀 포인트 (복사 후 수정 필요)
| 항목 | 설명 |
|------|------|
| `SYSTEM_PROMPT` | 서비스 성격에 맞는 어투·역할 정의 |
| `vectorDB 컬렉션` | 해당 프로젝트의 문서/FAQ/정책을 임베딩한 컬렉션으로 교체 |
| `LLM 모델` | 프로젝트 예산·성능 요건에 맞게 모델 선택 |
| `위젯 디자인` | 브랜드 컬러·아이콘 변경 |
| `환경변수` | API 키, DB 엔드포인트 등 |

#### 기술 스택 (예정)
- Next.js 14 API Route (백엔드)
- Vercel AI SDK (스트리밍)
- OpenAI Embeddings + Pinecone 또는 Supabase pgvector (RAG)
- React + Tailwind CSS (위젯 UI)

---

## 5. 비기능 요구사항

- **재사용성**: 모듈 복사 후 `.env` + 컨텍스트 파일만 교체하면 동작해야 함
- **독립성**: 각 모듈은 다른 모듈 없이 독립 실행 가능
- **문서화**: 커스텀 포인트를 모듈 README에 반드시 명시
- **보안**: API 키는 절대 하드코딩 금지, 환경변수로만 관리

---

## 6. 성공 지표

- 새 프로젝트에 모듈을 복사하고 **30분 이내**에 커스텀 완료 후 동작 확인 가능
- 모듈별 README만 읽어도 어떻게 커스텀할지 이해 가능

---

## 7. 제외 범위

- 모듈 간 통합 프레임워크 (각 모듈은 독립적으로 사용)
- npm 패키지로 배포 (단순 복사·붙여넣기 방식 유지)
- 자동화된 동기화 (원본 수정 시 복사본 자동 업데이트 없음)
