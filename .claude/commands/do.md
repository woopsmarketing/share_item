# /do 워크플로우

사용자 요청: $ARGUMENTS

## 실행 흐름

아래 순서로 반드시 실행한다.

### 1단계: spec-writer 호출

`spec-writer` 에이전트를 호출하여 위 요청을 작업 명세서로 변환한다.

전달할 내용:
- 사용자 요청 원문: "$ARGUMENTS"
- 현재 날짜: 오늘 날짜

### 2단계: 명세서 확인 및 승인

spec-writer가 출력한 명세서를 사용자에게 보여준다.

사용자에게 묻는다:
> "위 명세서대로 진행할까요? (Y/수정사항 입력)"

- **Y 또는 확인**: 3단계로 이동
- **수정 요청**: spec-writer를 다시 호출하여 수정된 명세서 생성

### 3단계: 추천 에이전트로 구현

명세서의 "추천 실행 에이전트" 항목을 확인한다.

- **단일 전문 에이전트** 추천 → 해당 에이전트 직접 호출 (명세서 전달)
- **복합 작업** 추천 (`implementer`) → `implementer` 에이전트 호출 (명세서 전달)

### 4단계: 완료 보고

CLAUDE.md의 "작업 보고서 출력 규칙" 형식으로 보고서 출력.
