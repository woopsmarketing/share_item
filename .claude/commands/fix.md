error-diagnoser 에이전트를 호출하여 현재 발생한 에러나 빌드 문제를 분석하고 수정한다.

## 실행 순서

1. 아래 컨텍스트 파일 먼저 읽기 (병렬):
   - `CURRENT_STATE.md` (있으면)
   - `package.json`

2. 에러 컨텍스트 파악:
   - `$ARGUMENTS` 가 있으면 해당 내용을 에러 힌트로 사용
   - 없으면 `pnpm build` 또는 `npm run build` 실행해서 에러 직접 확인

3. error-diagnoser 에이전트 호출:
   - 에러 메시지 전달
   - 관련 파일 목록 전달
   - "수정 후 빌드 재확인까지 완료" 조건으로 실행

## 입력

에러 내용 또는 문제 설명: $ARGUMENTS

에러 내용이 없으면 현재 프로젝트에서 빌드/린트 에러를 직접 조사한다.
